# Build Launch — Master Entry

**Purpose:** **Build Launch** is the named framework for building and launching the product, website, and voice agent **from every perspective** — with **full developer docs**, **executive-level** and **average-Joe** explanations, **technical** detail, **voice agent instructions** (conversations, troubleshooting), **website marketing launch sequence**, and **voice agent sequence** (core every-time + situational/variable). Top-level agents **further define and expand** core and situational instructions so that **Build Launch** is complete and consistent every time.

## What is Build Launch?

**Build Launch** = The end-to-end system to:

1. **Build** — Develop, configure, and verify the product (LitigationForce.AI, OneWish OS, voice agent, websites).
2. **Launch** — Execute the **website marketing launch sequence** and **voice agent sequence** so that go-live is consistent, compliant, and effective from **every perspective** (user, attorney, investor, partner, technical, compliance).
3. **Core every time** — A **core sequence** that runs every launch (checklists, deployment, disclaimer, voice agent first message, tough-Q handling).
4. **Situational / variable** — **Variable instructions** for different **audiences** (investor, executive, attorney, enterprise, compliance, technical, average Joe), regions, products, or channels; **expanded by top-level agents** so voice agents and marketing launch stay complete and on-message from **all perspectives**.

Top-level agents (orchestrator, platform-architect, marketing/launch specialists) **define and extend** Build Launch so that voice agents, website copy, launch steps, and **investor presentation slide(s)** are **fully specified** and **executable** every time — and so the same framework **makes sense in terms of every audience** discussed (investor, executive, attorney, enterprise, compliance, technical, average Joe).

## Expansion by top-level agents

**Core** instructions and sequences in this folder are the **minimum** that run every time. **Top-level agents** (orchestrator, platform-architect, and any agent designated to own “Build Launch” or “marketing launch”) must:

1. **Complete core** — Ensure every item in the core sequence is defined and executable (no placeholders for critical steps).
2. **Add situational/variable** — Define instructions and copy for each **perspective** (investor, executive, attorney, enterprise, compliance, technical, average Joe) and **variable** (region, product variant, channel), including **investor presentation slide(s)** and how Build Launch fits OneWish OS, NextTech OS, repos, agents, features, and functions so it **makes sense in terms of all perspectives**.
3. **Update voice agent** — Feed new instructions into the voice agent knowledge base and system prompt (ElevenLabs, agent-config, or RAG docs).
4. **Update website/marketing** — Add or refine launch steps, copy, and CTAs for each perspective so the **website marketing launch** is complete from every angle.

When agents add or change Build Launch content, they must **update this README** or the relevant doc and **sync** to voice agent config and launch runbooks so the system stays consistent.
