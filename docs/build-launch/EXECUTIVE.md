# Build Launch — Executive Summary

**Audience:** Executives, board, investors. One page.

---

## What is Build Launch?

**Build Launch** is our named framework for **building** and **launching** the product, website, and voice agent in a **repeatable, compliant way** from **every perspective** — user, attorney, investor, partner, technical, compliance. It ensures we don’t ship half-built or off-message; core steps run every time; situational and variable instructions are defined and expanded by our top-level agents.

---

## Why it matters

- **One playbook** — Product, website, and voice agent launch are defined in one place (Build Launch). No ad-hoc checklists or last-minute copy.
- **Every perspective** — We launch for attorneys, investors, enterprise, and compliance at once; Build Launch sequences and voice agent instructions cover each so we stay on-message and compliant.
- **Voice agent = front door** — “Talk to Alex” is often the first touch. Build Launch includes **voice agent instructions** (conversations, troubleshooting, escalation) so Alex answers consistently and hands off to humans when needed.
- **Top-level agents extend it** — Orchestrator and platform/marketing agents **expand** core and situational instructions so Build Launch stays complete as we add products, regions, or channels.

---

## Key milestones (Build Launch)

| Milestone | What happens |
|-----------|--------------|
| **Pre-launch** | Sites and voice agent configured; disclaimer live; Full Review and launch checklist done. |
| **Launch week** | Domains live (litigationforce.com, talk.litigationforce.com); blog/PR; voice agent linked in all CTAs. |
| **Post-launch** | Weekly/biweekly content; voice agent knowledge and escalation updated from real conversations; Build Launch docs updated by agents. |

---

## Voice agent and launch in 3 bullets

1. **Alex (voice agent)** holds conversations, answers tough questions (pricing, security, competitors), and escalates to Robert or the team per clear rules. Instructions live in Build Launch and in `marketing/voice-agent/agent-config.md`.
2. **Website marketing launch** runs in sequence: pre-launch content → launch week (sites + voice link + PR) → post-launch cadence. Build Launch doc defines the exact steps and pages.
3. **Core every time + situational** — Core sequence (disclaimer, first message, escalation) runs every launch; variable instructions (by audience, topic, channel) are expanded by top-level agents so we’re complete from every perspective.
